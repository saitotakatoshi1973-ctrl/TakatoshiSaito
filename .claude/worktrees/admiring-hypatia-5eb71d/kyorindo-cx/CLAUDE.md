# 杏林堂 CX推進部 戦略・ロードマップ検討プロジェクト

## プロジェクト概要

複数エージェントが最新情報をもとに、CX推進部の戦略とロードマップを継続的に検討・更新し、成果資料を作成するプロジェクト。

## フォルダ構成

| フォルダ | 用途 |
|----------|------|
| `00_organization/` | 組織図・組織情報 |
| `01_strategy/` | MVV・経営方針サマリー・ロードマップ（インプット） |
| `02_issues/` | 既存課題（課題/店舗生産性/オペレーション/データ基盤） |
| `03_progress/` | 週次業務進捗報告 |
| `04_references/` | 追加インプット（市場動向・競合・VoC・外部情報等） |
| `05_agents/` | エージェント定義ファイル |
| `06_outputs/` | 議論ログ・中間検討メモ |
| `07_deliverables/` | 完成した戦略資料・ロードマップ資料 |

## エージェント構成（予定）

- `agent_cxlead.md` - CX推進部長視点
- `agent_ax_dx.md` - AX/DX推進課視点
- `agent_strategy.md` - 経営企画視点

## 作業ルール

- インプット資料は `01_strategy/` 〜 `04_references/` に格納する
- 議論・検討の前に常に `01_strategy/` 〜 `04_references/` を読み直しする
- 議論・検討の途中成果は `06_outputs/` に保存する
- 正式な戦略資料・ロードマップは `07_deliverables/` に出力する
- 戦略資料・ロードマップは.pptxにまとめる
- 新しいカテゴリの資料は `04_references/` を受け皿として追加する

## ファイル処理方針

### Excelファイル（.xlsx）
- pandasとopenpyxlで読み込む
- 複数シートは全て確認する

### PDFファイル（.pdf）
- Claude APIのネイティブPDF機能を使う
- base64エンコードしてdocumentタイプで渡す
- スキャンPDF・図表を含む場合も同様に処理する
