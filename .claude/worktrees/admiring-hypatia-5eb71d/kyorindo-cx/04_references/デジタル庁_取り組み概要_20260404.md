# デジタル庁 取り組み概要 - 杏林堂コミュニケーション基盤・DX推進 参考資料

**作成日**: 2026年4月4日
**出所**: デジタル庁公式サイト・活動報告（2025年）・Microsoft顧客事例等
**活用目的**: 杏林堂のMS365中心コミュニケーション基盤・DX推進の参考

---

## デジタル庁 概要

| 項目 | 内容 |
|------|------|
| 設立 | 2021年9月 |
| 規模 | 約1,500人規模の混成体制（官民共同） |
| ミッション | 「誰一人取り残されない、人に優しいデジタル化」 |
| 方向性 | 行政DXから社会全体のDXへ（デジタル庁2.0） |

---

## 主要施策・成果（2025年時点）

### 1. マイナンバー関連

| 施策 | 状況 |
|------|------|
| マイナンバーカード保有率 | **約80%**（39%から拡大） |
| マイナ保険証 | 2025年3月開始 |
| マイナ免許証（デジタル免許証） | 2025年3月開始 |
| マイナポータル | 引越し・パスポート・子育て・介護等のオンライン手続き対応 |
| 今後の拡大 | オンライン死亡届、出生・子育て関連の一括申請、マイナポータルアプリとデジタル認証アプリの統合 |

### 2. ガバメントクラウド（Government Cloud）

| 指標 | 数値 |
|------|------|
| 利用システム数 | **4,892システム**（2025年2月時点）※2024年8月（671システム）比335%増 |
| 標準準拠システム移行 | 2025年度末までに全自治体の20業務を移行完了目標 |
| 対象 | 全国自治体の基幹業務（住民基本台帳・戸籍等20業務） |

### 3. ガバメントソリューションサービス（GSS）＝政府版MS365基盤

| 指標 | 内容 |
|------|------|
| 概要 | 政府共通の業務実施環境。ネットワーク・認証・セキュリティ・コミュニケーションを一元調達 |
| 導入機関 | **16行政機関**（2025年9月時点）、順次拡大中 |
| 接続ユーザー数 | **約46,400ユーザー**、約1,500拠点 |
| 主要ツール | **Microsoft 365**（Teams・SharePoint・OneDrive・Power Apps・Power Automate・Copilot） |
| セキュリティ | ゼロトラストネットワーク。不審メール自動隔離 |
| 成果 | テレワーク推進・柔軟な働き方・業務効率化を実現 |

#### GSSでのMS365活用事例
- **Teams**: リモート通話（高品質背景加工・ノイズキャンセリング）、現場写真・メモのリアルタイム共有
- **Power Apps / Power Automate**: 職員が手軽に業務自動化
- **Microsoft Copilot**: 業務効率の大幅向上
- **SharePoint / OneDrive**: 情報共有・リアルタイム連携

### 4. 生成AI活用

| 指標 | 内容 |
|------|------|
| 庁内生成AI利用 | 利用者950名、延べ65,000回（2025年時点） |
| 調達・活用ガイドライン | 2025年5月に「生成AI調達・活用ガイドライン」を策定 |
| 今後 | ガバメントAIを加速（2026年から本格導入） |

### 5. アナログ規制の見直し

|------|------|

### 6. 電子インボイス（JP PINT）

|------|------|

### 7. データ活用基盤

## デジタル庁2.0 の方向性

```
デジタル庁1.0（〜2025）    デジタル庁2.0（2026〜）
行政のデジタル化       →  社会全体のDXへ
各省庁個別対応        →  ガバメントAI・データ前提の運営
システム整備中心      →  政策立案とDXの両輪化
```

**6つの戦略的方向性**:
1. デジタルによる経済成長・生産性向上
2. 準公共分野のデジタル化（医療・教育・防災・子育て）
3. デジタルによる地域活性化
4. 誰一人取り残されないデジタル社会
5. デジタル人材育成
6. 国際戦略（DFFT：信頼性あるデータの越境流通）

## 杏林堂コミュニケーション基盤（MS365）検討への示唆

### GSSが証明したこと（政府規模での実証）

| 観点 | 内容 |
|------|------|
| **MS365は大規模展開で実証済み** | 政府46,400ユーザー・1,500拠点で稼働。ゼロトラストセキュリティも担保 |
| **現場でのTeams活用** | 外出先・現場からのリアルタイム情報共有が実現 |
| **Power Apps・Automateで業務自動化** | IT専門人材がいなくても職員自ら業務アプリ・自動化を構築 |
| **Copilotで業務効率化** | 議事録・文書作成・分析等の効率化を実証 |

### 杏林堂のスケジュールとの対比

| 時期 | デジタル庁 | 杏林堂 |
|------|-----------|--------|
| 2025年〜 | GSS本格展開・生成AI活用 | MS365検討・調達 |
| 2026年〜 | ガバメントAI本格導入 | MS365 本部員300名配布開始 |
| 2027年2月末 | - | MS365配布完了・Outlook契約終了 |
| 2027年〜 | - | Teams・Copilot等の本格活用開始 |

### 電子インボイス（JP PINT）への対応
デジタル庁が推進するJP PINT（Peppol準拠）は小売業の取引デジタル化に直結。ツルハグループ全体での対応方針を確認する必要あり。

### アナログ規制見直しの影響
医療・調剤分野（PMH・オンライン服薬指導等）のデジタル化が国レベルで加速。杏林堂の調剤DX対応は引き続き業界標準として維持が必要。
