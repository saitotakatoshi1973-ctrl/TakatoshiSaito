## ページ①：ネットワーク基幹構成

### 主要DC構成

| DC名 | 場所 | 役割 |
|------|------|------|
| KDDI DC（府中DC） | 府中 | 基幹DC。FW・UTM・プロキシ設置 |
| NEC DC（神戸DC） | 神戸 | 冗長DC。各種サーバ群 |
| 吉塚DC | 吉塚 | D11向けDC |

### セキュリティ構成
- **Zscalerサーバ**：ゼロトラストアクセス制御（KDDI DC内設置）
- **SOCサービス**：外部SOCによる監視
- **Sandbox**：不正通信の検査
- **UTM**：統合脅威管理
- **高頻度シグネチャ更新**：マルウェア対策

### 店舗接続構成
- **KDDI WVS（Wide Area Virtual Switch）**：NTT FLET'S経由含む広域網
- **LTEバックアップ**：5Mbps（回線障害時バックアップ）
- **メイン回線**：Act 50Mbps / Stb 50Mbps（1Gbps×3）

### 店舗内機器
| 機器 | 用途 |
|------|------|
| POSレジ | 販売端末 |
| PC | 業務端末 |
| iPad | 会社支給（プロキシ設定済み） |
| DENSO HT / KEYENCE HT | ハンディターミナル（在庫・検品） |
| デジタルサイネージ | 店内販促 |
| Beacon | 位置情報活用 |
| AIカメラ | 防犯・来客分析 |
| プリンター | 棚札・POP印刷等 |
| AP（アクセスポイント） | 無線LAN |

### プロキシ経由通信（直抜け対象外）
- R-SHIFT、リモネ、キビタン、ストマネ
- OBIC、iPad経由通信、プロキシ未設定端末
- 化粧品台帳、新CRM発（FEZ・TrueData・TIDE）への連携
- ブライセン、レデイ美容台帳、ECABINET（閲覧）
- 店番長、リテールフォース

### 調剤専用プロキシ
- 調剤系システムは専用プロキシ経由で接続

### HD情シス管理サーバ群（神戸DC内）

## 杏林堂固有のネットワーク情報（重要）

### 現状構成（2025年11月時点）

| 項目 | 内容 |
|------|------|
| 本社接続 | 杏林堂本社（元町オフィス）→ KDDI WVS、帯域30Mbps |
| 店舗・調剤接続 | KDDI L2網（他事業会社と異なる独自網） |
| 店舗帯域 | 10Mbps |
| NW特性 | デフォルトルートが他グループ会社と異なる |
| インターネットブレイクアウト | **杏林堂DCからインターネットへ直接ブレイクアウト済み**（ツルハHD経由ではない）。DCからのトラフィックは発生しない |
| LTEバックアップ | 杏林堂NWにバックアップ回線あり（80Mbps） |

### 🔄 進行中：ツルハHDとの NW統合プロジェクト

**2026年4月現在、杏林堂とツルハHDでネットワーク統合プロジェクトが進行中。**

#### サーバー移行
| 項目 | 内容 |
|------|------|
| 移行元 | 杏林堂DC内のサーバー群 |
| 移行先 | ツルハHD 神戸DC |
| 予定時期 | **2026年内完了予定** |

#### 店舗ネットワーク移行
| 項目 | 内容 |
|------|------|
| 変更内容 | 杏林堂店舗をKDDI L2網からKDDI WVS（ツルハグループ共通網）へ直接接続に変更 |
| 着手時期 | **2026年中旬着手** |
| 完了予定 | 着手から**1年間で全店移行** |

#### NW統合後の期待効果
- ツルハグループ共通のWVS利用により、グループシステムとの通信が効率化
- 神戸DCのグループ共通インフラ（ActiveDirectory・セキュリティ等）を活用可能
- 帯域・セキュリティ基準がグループ標準に統一される

---

> ⚠️ **ロードマップへの示唆**：NW統合完了後はグループ共通インフラの活用が可能になる。電子棚札・AIカメラ・リアルタイムデータ活用などのDX施策は、**NW統合スケジュール（サーバー移行：2026年内、店舗NW移行：2026年中旬〜2027年中旬）と連動して計画する必要がある**。統合前に先行導入する施策は、移行時の再設計コストを考慮すること。

### NW統合スケジュールとDX施策の関係（ロードマップ制約条件）

| 時期 | NW状況 | DX施策の方針 |
|------|--------|------------|
| 〜2026年内 | サーバー移行中 | 神戸DC移行後を前提とした施策設計を推奨 |
| 2026年中旬〜 | 店舗NW移行着手 | 移行済み店舗から順次DX施策を展開 |
| 2027年中旬〜 | 全店NW統合完了 | グループ共通インフラを前提とした本格展開 |

## グループ全体NWの特徴まとめ

| 特徴 | 内容 |
|------|------|
| 基幹網 | KDDI WVS（NTT FLET'S経由含む） |
| セキュリティ | Zscaler（ゼロトラスト）+ UTM + SOC + Sandbox |
| 冗長構成 | Act/Stb（アクティブ/スタンバイ）の2系統構成 |
| 店舗バックアップ | LTE回線（5Mbps）を全店舗に設置 |
| D11の独立性 | D11本社はゼロトラ端末のみ、吉塚DCを独自保有 |
| 杏林堂の独立性 | KDDI L2網（専用）+ デフォルトルートが独自 |
| 海外展開 | ベトナム・タイランド（接続設計作成中） |
