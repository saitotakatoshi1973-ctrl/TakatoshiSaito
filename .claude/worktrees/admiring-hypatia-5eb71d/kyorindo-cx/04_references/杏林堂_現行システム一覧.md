## BI / データ活用系

### 実績参照ツール KOGOORO（Dr.sum）

| 項目 | 内容 |
|------|------|
| システム名（通称） | KOGOORO |
| 製品名 | Dr.sum（ウイングアーク1st） |
| ETLツール | Pentaho |
| データソース | Vertica（ETLでデータ集計） |
| 主な機能 | 売上・粗利の予実対比（日別） |
| 粒度 | 店舗別 × 大分類別 × 日別 |
| 利用範囲 | **店舗PCまで公開**（本部員 + 全店舗） |

#### 補足
- 売上・粗利予算を店舗別・大分類別・日別に保持
- 実績データと日別で対比可能
- 店舗スタッフが自店の実績を確認できる運用

|------|------|

| 当初完成予定 | 2026年6月 |
| ローンチ予定 | **2026年8月**（データサーバーの神戸DC移設と合わせて） |

> **注意**: ETL改修はNW統合（DCサーバー移設）と密接に連動。移設スケジュールの遅延がKOGOOROローンチに影響する可能性あり。

### 見切り運用の比較

| 項目 | 杏林堂 | ツルハHD |
|------|--------|---------|
| レジ形態 | **フルセルフレジ**導入済み | フルセルフレジなし |

> **追加制約**: 端末切替時は**フルセルフレジとの連携方式の再設計**が発生し、**見切りシールのレイアウト変更**が必要となる。シールのバーコード形式・印字項目がフルセルフレジのスキャン仕様に依存するため、レジシステムとの整合確認が必須。

### シフト管理・タイムスケジュール作成

#### シフト作成フロー

| 対象 | サイクル | 内容 |
|------|---------|------|
| 本部員 | 月度 | OBICでシフト作成 |
| 店舗従業員 | 月度 | 月末に翌月分シフト作成 → 20日頃を目途にOBICへ登録 → 日々のタイムスケジュール作成 |
| 調剤店舗 | 月度 | 店舗従業員とほぼ同様のフロー |

#### タイムスケジュール（日々の作業スケジュール）作成方法

Excelマクロで運用中。作成コストを下げるために以下の仕組みを採用：

1. **曜日別標準シート**をテンプレートとして用意
2. マクロでテンプレートをコピー
3. **OBICの出勤状況を参照**して不要行を削除・セルを塗りつぶし
4. タイムスケジュール完成

> **特記**: OBICとExcelマクロの連携により、日々のスケジュール作成の手作業を削減している。ただし完全自動化ではなく、OBICのデータを参照しながらマクロが処理する半自動運用。

#### 調剤：応援依頼管理アプリ（開発中）

| 項目 | 内容 |
|------|------|
| 背景 | 調剤は人員が不足している店舗があり、余力のある店舗に応援を依頼する運用がある |
| 現状 | 応援依頼の調整を簡易化するアプリを**Excelマクロで作成中** |
| 用途 | 店舗間の応援要否・余力状況の把握・調整 |
| 位置づけ | 現場主導の課題解決（ツール化）の事例 |

> **示唆**: 調剤の人員調整は現在Excelマクロで内製対応中。将来的にはデジタル化・API連携による自動マッチングや、グループ内での横展開が検討対象となりうる。

#### 🔴 課題：シフト・タイムスケジュール作成の工数

シフト作成・タイムスケジュール作成は現状**工数がかかっており、改善ニーズが高い**。

| 項目 | 内容 |
|------|------|
| 課題 | シフト作成・タイムスケジュール作成に工数がかかっている |
| 現状 | Excelマクロによる半自動化（OBICの出勤状況を参照）で一定のコスト軽減はあるが不十分 |
| 検討方向 | **OBICと連携した専用アプリ、またはAIツールの導入**を検討したい |

**検討対象ツールの方向性**

| アプローチ | 概要 | 備考 |
|-----------|------|------|
| 勤怠連携型シフト管理アプリ | OBICのデータを直接参照・連携できる専用アプリ | OBIC対応製品の有無を確認要 |
| AIシフト最適化ツール | 過去実績・売上予測・スキルを考慮した自動シフト提案 | 市販のAIシフトSaaS（例：Shiftmation、シフオプ等） |
| MS365 Power Automate連携 | MS365基盤上でOBICのデータを取得し自動生成 | MS365配布（2027/2）後が現実的な実装タイミング |

> **ロードマップ上の位置づけ**: MS365配布完了（2027年2月）後に本格検討・実装が可能。OBICとの連携可否（API・データエクスポート仕様）の確認が前提条件。

### データ粒度（Vertica原本）

Verticaには各店舗の**レシートROWデータ**（1明細1行）が格納されており、JAN×店舗×トランザクション単位が最細粒度。MartSolution・KOGOROはともにこのVerticaから集計して表示。

### BIシステム機能比較

| 比較軸 | MartSolution | KOGOORO（Dr.sum） | 売上粗利日報 |
|--------|-------------|------------------|------------|
| データソース | Vertica | Vertica（Pentaho経由） | Vertica |
| 利用範囲 | 本部員のみ | 店舗PCまで（全社） | 店舗PCまで（全社） |
| 最小参照粒度 | **JAN × 店舗別** | **JAN × 店舗別** | **大分類まで** |
| 主な用途 | 経営BI・分析 | 売上・粗利予実対比（日別） | 売上粗利日報 |
| 導入範囲 | ツルハHD・杏林堂共通 | 杏林堂独自 | ツルハグループ共通 |
